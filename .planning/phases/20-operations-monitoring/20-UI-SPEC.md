# Phase 20 — UI 设计合约

> Phase 20 运营监控的视觉与交互合约。继承 Phase 17/18 建立的设计系统，由 planner/executor 消费。

---

## 设计系统

| 属性 | 值 |
|------|-----|
| 工具 | 无 |
| 预设 | 不适用 |
| 组件库 | fluent_ui |
| 图标库 | FluentIcons |
| 字体 | Segoe UI，回退到系统无衬线体 |

## 间距标度

声明值（必须是 4 的倍数）：

| Token | 值 | 用途 |
|-------|-----|------|
| xs | 4px | 状态指示器间距、内联图标间距、紧凑标签间距 |
| sm | 8px | 批次行内元素间距、按钮间隔、卡片内紧凑分组 |
| md | 16px | 默认控件间距、面板内边距、诊断卡片内部间距 |
| lg | 24px | 双区之间的分隔间距、节标题下方间距 |
| xl | 32px | 页面顶部/底部呼吸间距、监控区整体内边距 |
| 2xl | 48px | 空状态垂直节奏 |
| 3xl | 64px | 大型空状态顶部/底部间距 |

例外：无

## 字体排版

| 角色 | 大小 | 字重 | 行高 |
|------|------|------|------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.3 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.15 |

字体规则：
- 仅使用上述四个字号。
- 仅使用 `400` 和 `600` 两个字重。
- 状态标签、指标数值、时间戳使用 `Label`。
- 区域标题（"批次任务监控"、"Sidecar 诊断"）使用 `Heading`。
- 空状态主标题使用 `Display`。

## 配色

| 角色 | 值 | 用途 |
|------|-----|------|
| 主色 (60%) | `#F5F7FB` | 监控页背景、批次列表画布、诊断区背景 |
| 辅助色 (30%) | `#E7ECF5` | 状态卡片表面、批次行悬停底色、导航面板表面 |
| 强调色 (10%) | `#2563EB` | 进行中状态指示、WebSocket 连接活跃指示、选中批次行边框、重启按钮 |
| 语义-成功 | `#16A34A` | 就绪状态（ready）、完成状态（completed） |
| 语义-警告 | `#D97706` | 降级状态（degraded）、部分失败批次 |
| 语义-危险 | `#C42B1C` | 停止状态（stopped）、失败状态（failed）、重启确认对话框危险强调 |

强调色保留用于：进行中状态指示器、WebSocket 连接活跃指示、选中批次行高亮边框、重启操作按钮 — 绝非所有交互元素。

语义色规则：
- 成功/警告/危险三色仅用于状态语义指示，不用于装饰或普通交互。
- 状态标签使用语义色填充底色 + 白色文字的组合标签样式（不是纯色圆点）。
- Sidecar 状态卡片的顶部色带使用对应的语义色（3px 顶部边框或顶部色条）。

## 视觉层次

- **首要焦点：** 批次任务监控区的批次列表。管理员进入页面后首先看到的是当前批次概览，因为批次任务是日常运营的核心。
- **次要焦点：** Sidecar 诊断状态卡片。状态卡片位于下半区顶部，以色带+大字号状态值快速传达基础设施健康。
- **三级焦点：** 点击下钻后的任务明细面板。仅在管理员主动点击批次行后出现。
- 错误摘要列表和队列指标作为辅助信息，视觉权重低于状态卡片和批次列表。

## 交互合约

### 导航入口
- 在 `fluent_app_shell.dart` 的 NavigationPane items 中添加第 6 项，图标为 `FluentIcons.diagnostic`，标题为"运营监控"。
- `NavigationProvider` 增加 `operationsIndex = 5`，`itemCount` 更新为 6。
- 监控页沿用 `_ShellPage` 包裹模式（顶部工具栏 + 页面内容），与 Phase 17/19 一致。

### 单页双区布局
- 页面垂直滚动，上半区为批次任务监控，下半区为 Sidecar 诊断面板。
- 两区之间使用 `lg`（24px）间距 + 水平分割线分隔。
- 每个区域使用 `xl`（32px）内边距。
- 批次列表区域无固定高度限制，随批次数量自然伸展。
- Sidecar 诊断区域最小高度 `320px`，确保状态卡片始终可见。

### 批次任务监控区
- 区域标题："批次任务监控"（Heading 级别）。
- 批次列表为垂直行列表，每行展示：状态标签（带语义色的紧凑标签）、批次名称/ID、进度信息（百分比文本 + 线性进度条）、时间信息（创建时间/完成时间）、错误计数。
- 默认按创建时间倒序排列。
- 点击批次行展开/导航到任务明细层：显示该批次下所有任务的状态、错误原因、重试提示。
- 任务明细层以行列表形式展示，复用 Phase 14 的 grouped failure reasons 契约。
- 线性进度条宽度 100%，高度 4px，使用强调色 `#2563EB` 填充已完成部分。

### Sidecar 诊断面板
- 区域标题："Sidecar 诊断"（Heading 级别）。
- 顶部为状态卡片：
  - 左上角大字号状态值（"就绪" / "降级" / "已停止"），使用对应语义色。
  - 状态卡片顶部 3px 色带使用对应语义色。
  - 右侧展示运行时长、最后探测时间（Label 字号）。
- 卡片内嵌入"重启 Sidecar"按钮（outlined 样式），仅在状态为"降级"或"已停止"时启用。
- 状态卡片下方为 Go 运行时指标行：队列深度、活跃 Worker 数、待处理任务数。使用 2-3 个紧凑指标卡片横向排列。
- 再下方为最近错误摘要列表：每条显示错误类型（Label 加粗）、时间戳、简短消息。最多显示最近 10 条。
- 错误列表为空时显示"近期无错误记录"。

### 重启操作
- 点击"重启 Sidecar"按钮后弹出确认对话框。
- 对话框标题："确认重启 Sidecar"。
- 对话框内容："重启将中断正在进行的 {N} 个计算任务。确定要继续吗？"
- 对话框操作按钮："取消"（次级）+ "确认重启"（危险样式，使用语义-危险色 `#C42B1C`）。
- 重启进行中时按钮显示加载状态并禁用。

### WebSocket 连接
- 进入监控页时自动建立 WebSocket 连接。
- WebSocket 连接状态在页面右上角以小标签形式展示："已连接"（成功色）/ "已断开"（危险色）。
- 连接断开时显示断连提示横幅："实时连接已断开，数据可能不是最新。" + "重新连接"按钮。
- 离开监控页时自动断开 WebSocket。

### 边缘状态
- 监控数据加载中：批次区域和诊断区域各自显示骨架屏/加载指示器。
- Go 后端不可达：整个页面显示"服务暂时不可用" + "检查后端连接后重试" + "重试"按钮。
- Sidecar 停止但 Go 正常：批次监控区正常展示，诊断区显示"已停止"状态卡片 + 错误摘要 + 重启按钮。
- 无批次数据：批次区域显示空状态（见文案合约）。

## 文案合约

| 元素 | 文案 |
|------|------|
| 导航项 | 运营监控 |
| 批次区域标题 | 批次任务监控 |
| 诊断区域标题 | Sidecar 诊断 |
| 主要操作按钮 | 重启 Sidecar |
| 空状态标题 | 暂无导入任务 |
| 空状态正文 | 当有图片导入任务提交后，批次和处理状态将在此处展示。 |
| 服务不可用 | 服务暂时不可用。检查后端连接后重试。 |
| WebSocket 断连提示 | 实时连接已断开，数据可能不是最新。 |
| WebSocket 重连按钮 | 重新连接 |
| 重新加载按钮 | 重试 |
| 批次状态-等待中 | 等待中 |
| 批次状态-进行中 | 进行中 |
| 批次状态-已完成 | 已完成 |
| 批次状态-失败 | 失败 |
| Sidecar 状态-就绪 | 就绪 |
| Sidecar 状态-降级 | 降级 |
| Sidecar 状态-已停止 | 已停止 |
| 无错误记录 | 近期无错误记录 |
| 破坏性确认标题 | 确认重启 Sidecar |
| 破坏性确认内容 | 重启将中断正在进行的 {N} 个计算任务。确定要继续吗？ |
| 破坏性确认取消 | 取消 |
| 破坏性确认执行 | 确认重启 |

附加文案规则：
- 进度百分比显示格式：`{N}%`。
- 时间戳格式：`YYYY-MM-DD HH:mm`。
- 错误摘要中的错误类型使用原始错误分类名称。
- WebSocket 连接状态标签："已连接" / "已断开"。

## 无障碍合约

- 批次行点击目标区域最小高度 `48px`，满足触控友好标准。
- 状态标签使用语义色底色 + 文字双重编码，不仅依赖颜色传达状态。
- 进度条提供文字百分比替代信息（`aria-label` 语义标注或可见文本）。
- 重启确认对话框可通过键盘 Escape 关闭。
- WebSocket 连接状态标签提供文字描述，非仅图标。
- 错误摘要列表中的时间戳和错误消息保持可读对比度。
