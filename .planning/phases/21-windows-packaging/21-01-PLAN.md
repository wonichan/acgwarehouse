<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| launcher/bundle root → Go bootstrap | Untrusted filesystem paths and env values influence child-process startup |
| Go bootstrap → Python sidecar | Command-line arguments and runtime files cross process boundary |
| diagnostics/log files → user/operator | Failure output is user-visible and must not leak secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01 | T | portable runtime path resolution | mitigate | Resolve child paths relative to bundle root only; reject empty/unknown component values in diagnostic output |
| T-21-02 | I | startup diagnostics file | mitigate | Store only component, message, and log file paths; never include admin credentials or full request payloads |
| T-21-03 | S | sidecar executable selection | mitigate | Prefer explicit `ACG_SIDECAR_EXECUTABLE` in packaged mode and verify file existence before launch |
| T-21-04 | D | sidecar startup | mitigate | Fail fast with classified diagnostic output and bounded startup timeout rather than silent retries |
</threat_model>
