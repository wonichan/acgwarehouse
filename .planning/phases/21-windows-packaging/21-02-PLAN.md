## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user launch → Flutter bootstrap | Executable location and bundle contents are attacker-controllable filesystem inputs |
| Flutter bootstrap → Go child process | Environment variables and executable paths control startup behavior |
| startup diagnostics → failure screen/dialog | User-visible failure output must remain helpful but not secret-bearing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-21-06 | T | startup error surface content | mitigate | Read only the structured diagnostic file and whitelist component values before rendering dialog/screen text |
| T-21-07 | D | packaged shutdown flow | mitigate | Use bounded wait + kill fallback so orphaned Go processes do not persist after UI exit |
</threat_model>
