    1. Create `services/python-sidecar/compute/hashing.py`:
       - `compute_image_hashes(image_path: str) -> dict`: Computes both SHA256 and 256-bit pHash for a single image. Returns `{"path": str, "sha256": str|None, "phash": str|None, "error": str|None}`. Uses `hashlib.sha256()` for file hash (8192-byte chunks). Uses `imagehash.phash(Image.open(path), hash_size=16)` for 256-bit pHash, converts via `str(phash)` → 64-char hex. Wraps in try/except, sets error on failure.
       - `batch_compute_hashes(image_paths: list[str], max_workers: int = 4, progress_callback=None) -> list[dict]`: Uses `concurrent.futures.ThreadPoolExecutor(max_workers=min(cpu_count, max_workers))`. Submits all paths, collects via `as_completed`, calls `progress_callback(percent)` after each. Closes PIL Image objects after hashing. Returns list of result dicts.
       - Use `pathlib.Path` for all path handling (per Pitfall 4 — Windows paths).
       - Close PIL Image after computing hash to avoid memory pressure (per Pitfall 6).

    2. Create `services/python-sidecar/compute/grouping.py`:
       - `class UnionFind`: `__init__(n)`, `find(x)` with path compression, `union(x, y)` with union by rank, `groups() -> dict[int, list[int]]` returns only groups with >1 member.
       - `hamming_distance(hash1: str, hash2: str) -> int`: Uses `imagehash.hex_to_hash(hash1) - imagehash.hex_to_hash(hash2)` (per research — do NOT hand-roll XOR on hex strings).
       - `group_duplicates(hashes: list[dict], threshold: int) -> list[dict]`: Takes list of `{"image_id": int, "phash": str|None, "sha256": str|None, ...}`. First groups by sha256 (exact duplicates). Then uses Union-Find for O(n²) pairwise hamming distance comparison on remaining images with valid phash. Returns list of `{"group_id": int, "member_indices": list[int], "type": "exact"|"similar"}`. Per Pitfall 3: O(n²) is acceptable for <20k images.

    3. Create `services/python-sidecar/compute/scoring.py`:
       - `FORMAT_PRIORITY = {"png": 1.0, "webp": 0.9, "jpeg": 0.7, "jpg": 0.7, "gif": 0.5, "bmp": 0.6}` (per D-09, D-11).
       - `compute_recommendation_score(width: int, height: int, file_size: int, format: str) -> tuple[float, list[dict]]`: Returns `(composite_score, reasons)`. Factors:
         - Resolution (weight 0.5): `min(100, (width*height / 8_294_400) * 100)` — 4K=100
         - File size (weight 0.3): `min(100, (file_size / 10_485_760) * 100)` — 10MB=100
         - Format (weight 0.2): `FORMAT_PRIORITY.get(format.lower(), 0.5) * 100`
         - Each reason: `{"factor": str, "value": str, "score": round(score, 1), "weight": float}`
         - Composite: `sum(r["score"] * r["weight"] for r in reasons)`, rounded to 1 decimal (per D-10).
       - `select_recommended(members: list[dict]) -> tuple[int, float, list[dict]]`: Takes list of member dicts with width/height/file_size/format. Returns `(recommended_index, score, reasons)` for the highest-scoring member.
