# PRD: 图片详情页操作刷新与图库滚动位置记忆

## 背景

用户反馈两个前端体验问题：

1. 在图片详情页执行打分或收藏操作后，页面会整体刷新一下（加载骨架屏闪现），体验割裂。
2. 从详情页返回图库页时，之前浏览的图片位置丢失，需要从头往下滑动重新找。

## 目标

- 详情页打分/收藏/标签操作后，不再出现整页加载闪烁，数据仍以服务端为准。
- 从详情页返回图库页时，恢复离开时的滚动位置和已加载的瀑布流内容。

## 范围

- 仅前端 `frontend/vue-gallery`，不涉及后端改动。
- 涉及文件：`src/pages/DetailPage.vue`、`src/pages/GalleryPage.vue`、`src/App.vue`。

## 不做

- 不改后端 API 契约或返回结构。
- 不对搜索页/热榜页/收藏页做 KeepAlive（仅图库页）。
- 不引入 Pinia 或全局状态管理库。
- 不改图库筛选/分页的加载逻辑。

## 约束

- 遵循 `api-client.md` spec：mutation 成功后详情页最终状态必须以重新 `getImage(id)` 为准，不能纯本地伪造。
- 遵循 `component-guidelines.md`：瀑布流列布局不能因 KeepAlive 恢复而错乱。
- `verbatimModuleSyntax`：类型导入必须用 `import type`。
- 响应式布局不能因 KeepAlive 破坏。

## 验收标准

### AC1: 详情页打分不闪烁
- 登录后在详情页点击"保存评分"，页面不出现加载骨架屏，评分按钮短暂显示"保存中..."后恢复，"我的评分"标签立即更新为新分数，`avg_score`/`rating_count` 在后台静默刷新后更新。
- 评分失败时显示 toast 错误，不闪烁。

### AC2: 详情页收藏不闪烁
- 登录后在详情页选择收藏夹并添加成功后，页面不出现加载骨架屏，"已收藏"标签出现，`favorite_count` 静默刷新后更新。
- 收藏失败时 picker 保持打开并显示错误，不闪烁。

### AC3: 详情页标签操作不闪烁
- 登录后在详情页添加/移除标签成功后，页面不出现加载骨架屏，标签列表静默刷新后更新。
- 失败时 picker 保持打开并显示错误，不闪烁。

### AC4: 图库滚动位置恢复
- 在图库页向下滚动若干屏 → 点击某张图片进入详情页 → 点击浏览器返回/RouterLink 返回图库页 → 滚动位置恢复到离开时的位置，已加载的瀑布流卡片仍在。
- 不重新发起 `page=1` 的首屏请求。

### AC5: 图库功能不退化
- 图库筛选切换仍正常重新加载。
- 无限滚动 sentinel 在返回后仍能触发下一页加载。
- 窗口 resize 仍能触发瀑布流列数重算。
- 每日推荐和社区精选在首次加载时正常工作。

### AC6: 构建通过
- `npm run build`（含 `vue-tsc -b`）通过，无类型错误。
