# 个人中心新增签到功能

## Goal

在个人中心（AccountPage）新增签到功能：用户每日首次访问时自动签到并获得积分，通过签到日历框可视化展示当月签到记录与累计积分。全程无需手动点击，不支持补签。

## Requirements

### 功能需求

- **自动签到**：用户每日首次登录/访问个人中心时，系统自动完成当日签到，无需手动点击按钮。
- **签到奖励**：每次签到获得 10 积分，积分累计存储于用户账户。
- **签到日历框**：个人中心页面新增一个日历框，可视化展示当月签到记录（已签到日期高亮）。
- **积分展示**：日历框内展示当前累计积分总数。
- **月份切换**：日历框支持切换至上月/下月，查看历史签到记录。
- **幂等性**：同一日重复登录或访问，不重复签到、不重复加积分。
- **不支持补签**：不提供任何补签入口或手动签到按钮，错过的日期无法补登。

### 约束

- 后端遵循 `handler -> service -> ports -> repository` 分层，DO 为纯值类型，业务规则在 service 层。
- 前端遵循 Vue 3 + TypeScript + composables 模式，类型导入使用 `import type`。
- 签到日期以亚洲/上海时区（UTC+8）计算"今日"，保证中国大陆用户跨午夜签到正确。
- 数据库为 SQLite，新表通过 GORM AutoMigrate 自动创建。
- 自动签到触发点为 `GET /users/me`（个人中心加载与每次会话恢复均调用此接口），覆盖"每日首次登录"语义——因 JWT 有效期 7 天，用户每日访问不必然调用 `/users/login`，以 `/users/me` 触发可保证每日首访即签到。

## Acceptance Criteria

- [ ] 用户当日首次访问个人中心后，系统自动完成签到，积分 +10。
- [ ] 同日再次访问或登录，积分不变，签到记录不重复。
- [ ] 个人中心页面展示签到日历框，当月已签到日期可视化高亮。
- [ ] 个人中心展示当前累计积分总数。
- [ ] 日历框可切换至上月/下月查看历史签到记录。
- [ ] 不存在任何补签入口或手动签到按钮。
- [ ] 后端单元测试覆盖：首签加分、重复签到幂等、月份查询。
- [ ] 前端 `npm run build` 类型检查与构建通过。
- [ ] 后端 `go build ./...` 与 `go test ./...` 通过。

## Notes

- 签到日历框仅展示，无交互签到动作（自动签到）。
- 积分当前仅来自签到，后续若扩展其他积分来源，积分字段已为累计值，无需改动存储结构。
