# 每日推荐

## Goal

新增“每日推荐”功能，为首页或推荐区域提供一组每日更新的随机图片推荐。它必须与现有“社区精选/热榜”区分：社区精选来自热榜算法与行为指标；每日推荐来自随机选择机制，但随机过程需要具备公平性，尽量保证所有可展示图片最终都有机会被抽中。

## Background / Confirmed Facts

- 项目是单仓库，后端 Go + Hertz + GORM + SQLite，前端 Vue 3 + TypeScript + Vite；接口统一走 `/api/v1`。
- 现有首页 `frontend/vue-gallery/src/pages/GalleryPage.vue` 会并行加载：
  - `getImages(...)` 用于图库瀑布流；
  - `getRankings({ period: 'week', limit: 20 })` 用于“本周社区焦点/社区精选轮播”。
- 现有前端 API 客户端 `frontend/vue-gallery/src/api/client.ts` 的 `getRankings` 调用 `/rankings`，并返回热榜缓存列表。
- 现有轮播组件 `frontend/vue-gallery/src/components/Carousel.vue` 的文案和无障碍标签固定指向“本周社区焦点 / 社区精选轮播”。如果每日推荐也用轮播，需要抽象或新增组件文案能力。
- 现有后端热榜由 `internal/job/ranking_job.go` 定时重算，`internal/repository/ranking.go` 通过行为事件聚合并缓存到 `ranking` 表；这不应被每日推荐复用为算法来源。
- 可公开展示图片以 active 状态为准：`image.status = active` 且 `deleted_at IS NULL`；相关查询封装在 `internal/repository/image.go`。
- SQLite 自动迁移入口在 `internal/infra/db/sqlite.go` 的 `AutoMigrate`，当前包含 `po.Ranking` 等持久化对象。
- 现有路由测试覆盖 `/api/v1/rankings` 的合法/非法 period 行为，可作为新增每日推荐路由测试风格参考。

## Requirements

- R1. 新增每日推荐能力，语义上与社区精选/热榜隔离，不改变现有 `/api/v1/rankings` 的行为。
- R2. 每日推荐只从可公开展示的 active 图片中选择，不返回软删除或不可展示图片。
- R3. 每日推荐的选择应是随机的，但不是每次请求即时全量随机；应通过公平策略减少重复，尽量让所有 eligible 图片在长期运行中都能进入推荐。
- R4. 每日推荐结果应具备稳定的“每日”语义：同一天内刷新页面不应频繁变动，除非产品决策另有要求。
- R5. 前端需要把“每日推荐”与“社区精选”清晰区分，避免用户误以为每日推荐来自热榜算法。
- R6. 空数据或图片 URL 不可展示时，前端需要有稳定降级，不影响图库列表正常加载。
- R7. 新功能需要有后端公平选择/持久化测试、HTTP 路由测试，以及前端构建/typecheck 验证。

- R8. 每日推荐采用全站统一结果：同一自然日内，所有用户和匿名访客看到同一组每日推荐。
- R9. 每日推荐默认每天展示 10 张图片。
- R10. “每日”的日期边界使用 Asia/Shanghai（北京时间）自然日。
- R11. 前端首页新增独立“每日随机推荐”区块，与现有社区精选/热榜区块并存，不替换现有社区精选。
- R11a. 每日推荐前端展示以图片为主，只保留必要辅助文字；不展示无必要的 inline note、时区字符串或刷新机制说明。
- R12. 本期只提供“今日推荐”，不提供面向用户的历史每日推荐查询。
- R13. 新增或恢复的 active 图片进入当前随机周期的剩余池；软删除图片从剩余池和新生成的今日结果中过滤；当剩余池不足 10 张时，开启新一轮全量洗牌周期补齐。

## Acceptance Criteria

- [ ] `GET /api/v1/rankings` 现有热榜/社区精选接口和行为保持兼容。
- [ ] 新增每日推荐接口返回固定数量以内的 active 图片，响应格式遵循现有 API envelope/list 风格。
- [ ] 同一自然日内多次请求每日推荐返回稳定结果。
- [ ] 当推荐池足够大时，连续多个北京时间自然日不会只重复同一小批图片；已推荐图片会在一个随机周期内被延后，直到池中其他图片也有机会被选中。
- [ ] 新增或恢复的 active 图片进入当前随机周期的剩余池，有机会在当前周期被抽中。
- [ ] eligible 图片发生软删除后，不再出现在新的每日推荐结果中。
- [ ] 前端首页展示每日推荐时，文案明确为随机/每日推荐，不复用“社区精选/热榜”语义。
- [ ] 后端 `go test ./...` 通过。
- [ ] 前端 `npm run build` 在 `frontend/vue-gallery` 通过。

## Out of Scope

- 不提供按日期查看历史每日推荐的用户界面或公开查询接口。
- 不重写现有热榜、评分、收藏或浏览事件算法。
- 不引入个性化用户画像推荐，除非后续明确要求。
- 不改变图片同步、搜索索引或详情页浏览计数逻辑。
- 不在本阶段承诺后台管理配置界面。

## Decisions

- D1. 每日推荐采用全站统一的一组结果，而不是按用户或访客个性化。
- D2. 每日推荐每天展示 10 张图片。
- D3. “每日”的日期边界使用 Asia/Shanghai（北京时间）自然日。
- D4. 前端首页采用独立“每日随机推荐”区块，展示 10 张卡片/小网格，并与现有社区精选并存。
- D4a. 原型去掉 inline note；前端展示遵循图片为主、必要文字辅助。
- D5. 本期只做今日推荐，不提供按日期查看历史每日推荐。
- D6. 新增/恢复的 active 图片进入当前随机周期的剩余池；软删除图片从剩余池和新生成结果中过滤；剩余池不足 10 张时开启新一轮全量洗牌周期补齐。
