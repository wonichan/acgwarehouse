# 重设计本周社区焦点轮播

## Goal

改善图库首页“本周社区焦点 / 社区精选轮播”的阅读与操作体验，使轮播卡片在桌面、平板、移动端均清晰可用，并与页面背景形成足够层次。

## Confirmed Facts

- 入口组件为 `frontend/vue-gallery/src/components/Carousel.vue`，由 `frontend/vue-gallery/src/pages/GalleryPage.vue` 使用。
- 数据来自 `getCommunityFocusSlides()`，后端图片响应已有 `width` / `height`，本次不改接口和排序，但需把宽高透传到轮播展示类型。
- 现有全局样式集中在 `frontend/vue-gallery/src/assets/app.css`，但前端规范要求组件专属样式优先放在组件 `<style scoped>` 中。
- 现状问题来自截图与代码：图片与文字间距不足、标题/文件名过长时抢占视线、底部横向导航拥挤、移动/平板布局不足、卡片与页面浅色背景层次不清。

## Requirements

- 设计执行需参考 `redesign-existing-projects`、`high-end-visual-design`、`design-taste-frontend-v1`，但以本项目 Vue + CSS token 体系为准，不迁移框架，不引入新依赖。
- 重新组织轮播卡片的信息层级：图片、标签、序号、标题、描述、热度/收藏数据应有明确间距和主次。
- 轮播不直接展示文件名；项目重点是图片本身，文件名等详细信息通过点击图片进入图片详情页获取。
- 图片需可点击进入详情页，目标路由为 `/detail?id=<imageId>`。
- 轮播图片区域需依据后端返回的图片宽高适配不同宽高比，不能把所有图片强行裁成同一固定比例；宽高缺失或异常时才使用安全默认比例。
- 底部滑动条改为清晰的进度/分页导航，不应出现截图中拥挤、割裂、难点击的问题。
- 桌面端保留图文并排；平板端允许压缩比例但不得重叠；移动端改为上下布局，按钮和分页项保持可触达。
- 轮播外层与内部卡片需从背景中分离，使用现有色彩变量、边框、阴影、留白和响应式约束。
- 保留键盘左右键、上一张/下一张、点击分页跳转、ARIA 文案等现有交互语义。

## Acceptance Criteria

- [ ] 轮播在桌面宽屏下图文间距清楚，图片不挤压文案，标题和数据不溢出。
- [ ] 轮播主内容不展示文件名，文案聚焦排名、热度、评分、收藏等社区焦点信息。
- [ ] 点击轮播图片可进入对应图片详情页。
- [ ] 不同宽高比图片在轮播中按自身比例展示，超高/超宽图片有合理边界，不撑破页面或被粗暴裁切。
- [ ] 轮播在平板宽度下布局自然换行或压缩，无横向页面滚动。
- [ ] 轮播在移动宽度下改为单列，底部导航与按钮可点击，文字不遮挡其他内容。
- [ ] 底部导航能表达当前位置，并支持点击切换；视觉上比原横向 chip 滚动更清楚。
- [ ] 外层面板和焦点卡片与页面背景有足够对比，不再“融为一体”。
- [ ] `npm run build` 在 `frontend/vue-gallery` 通过，或明确记录阻塞原因。

## Out of Scope

- 不改后端接口。
- 不改轮播数据生成逻辑。
- 不新增第三方 UI 或轮播库。
- 不重做图库首页其他区域。
