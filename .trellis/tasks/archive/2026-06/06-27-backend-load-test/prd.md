# 后端服务压测与并发检查

## Goal

设计并执行压测用例，在 2c2g 云服务器环境下评估后端服务性能表现，输出测试报告并检查是否存在并发问题。

## Requirements

### REQ-1: 压测用例设计
- 覆盖核心 API 路径：图片列表、详情、搜索、评分、收藏、热榜
- 模拟真实用户行为：混合读写比例
- 适配 2c2g 硬件约束（wrk 线程数 ≤ 4）

### REQ-2: 并发问题检查
- 检测 SQLite 写竞争导致的超时/死锁
- 检测 ViewBuffer 高频写入时的阻塞和数据丢失
- 检测并发评分/收藏的数据一致性
- 检测搜索索引 Bleve 并发访问异常
- 检测热榜任务与用户请求的 writeDB 竞争

### REQ-3: 性能基准
- 建立 QPS、响应时间、错误率基线
- 识别性能瓶颈点（数据库、内存、磁盘 I/O）
- 资源使用监控（CPU、内存、磁盘 I/O）

### REQ-4: 测试报告
- 压测结果汇总（QPS、P50/P95/P99 延迟、错误率）
- 并发问题清单（如有），包含问题位置及严重程度
- 性能优化建议

## Acceptance Criteria

- [ ] 压测脚本可执行，覆盖至少 5 个核心 API
- [ ] 测试报告包含 QPS、P50/P95/P99 延迟、错误率
- [ ] 明确结论：是否存在并发问题，问题位置及严重程度
- [ ] 针对发现的问题给出修复建议
- [ ] 测试数据已清理，数据库恢复正常

## Constraints

- **压测工具**：wrk + Lua 脚本
- **测试环境**：本地 4c4g 云服务器，按 2c2g 约束压测，后端服务端口 7070
- **测试数据**：图片数据已存在，需模拟评分/收藏/用户数据，测试完成后删除
- **数据隔离**：所有测试数据使用 `loadtest_` 前缀
