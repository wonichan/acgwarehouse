# 命名与代码风格规范

AI 生成的所有代码必须符合 Go 官方社区标准及以下团队特定的命名规则。

## 1. 命名矩阵
| 类型         | 命名规则                              | 正确示例                    | 反例 (禁止)                   |
| :----------- | :------------------------------------ | :-------------------------- | :---------------------------- |
| **Go 文件**  | 全小写，单词间用 `_` 分隔             | `flow_task.go`              | `flowTask.go`, `flow-task.go` |
| **测试文件** | 以 `_test.go` 结尾，与被测文件同名    | `user_test.go`              | `test_user.go`                |
| **普通变量** | 驼峰式，首字母小写                    | `userCount`                 | `user_count`                  |
| **全局变量** | 驼峰式，首字母大写                    | `MaxRetries`                | `MAX_RETRIES`                 |
| **常量**     | 驼峰式，首字母大写                    | `DefaultTimeout`            | `DEFAULT_TIMEOUT`             |
| **普通函数** | 驼峰式，首字母大写(导出) / 小写(私有) | `GetUser()` / `parseData()` | `get_user()`                  |
| **Bool函数** | 必须使用 `Is`/`Has`/`Can` 等前缀      | `IsValid()`                 | `checkValid()`                |
| **构造函数** | 必须使用 `New` 前缀                   | `NewClient()`               | `InitClient()`                |
| **结构体**   | 驼峰式                                | `UserProfile`               | `user_profile`                |

## 2. 代码风格硬性要求
- **缩进**：必须使用 **Tab 字符** 进行缩进，禁止使用空格缩进。
- **符号空格**：运算符前后、逗号（`,`）后面必须加空格。
- **行宽限制**：单行代码长度绝对**不能超过 120 个字符**。超过时必须进行优雅换行。
- **格式化工具**：代码生成后，必须使用 `gofmt -s` 进行自动格式化。
