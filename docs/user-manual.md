## 系统概述

ACGWarehouse 是一个面向二次元爱好者的本地图片库管理系统。它帮助用户高效地管理和检索大量二次元图片，通过 AI 自动化减少手动整理的工作量，实现"存入即整理"的体验。

### 核心功能

- **AI 自动标签**: 调用千问/豆包等多模态 AI 为图片生成描述性标签
- **标签治理**: 将 AI 生成的原始标签归并为标准标签，支持别名和近义表达
- **重复检测**: 基于 SHA256 文件哈希和感知哈希检测重复/相似图片
- **智能搜索**: 支持按文件名、标签、组合标签搜索
- **收藏夹管理**: 创建和管理图片收藏夹/相册
- **批量操作**: 对多张图片进行批量标签、移动、删除操作
- **管理后台**: Web 界面监控系统状态和任务队列

### 技术栈

- **后端**: Go 1.23 + Gin + SQLite
- **前端**: Flutter Web
- **AI 服务**: 千问/豆包（OpenAI 兼容 API）
- **部署**: Docker Compose

## 快速开始

### 前提条件

- Docker Engine 20.10+
- Docker Compose v2.0+
- 约 2GB 可用磁盘空间

### 安装步骤

#### 1. 创建目录结构

```bash
mkdir -p acgwarehouse && cd acgwarehouse
mkdir -p data library deploy/config
```

目录说明：
- `data/`: SQLite 数据库文件（自动创建）
- `library/`: 图片库目录
- `deploy/config/`: 配置文件目录

#### 2. 下载配置文件

复制示例配置文件：

```bash
cp deploy/config/config.example.yaml deploy/config/config.yaml
```

#### 3. 配置 AI 服务

编辑 `deploy/config/config.yaml`：

```yaml
server:
  host: "0.0.0.0"
  port: 8080

ai:
  provider: "qwen"  # 或 "doubao"
  api_key: "your-api-key-here"  # 替换为你的 API 密钥
  model: "qwen-vl-max"
  requests_per_minute: 60

storage:
  scan_roots:
    - "/library"
```

#### 4. 启动服务

```bash
docker compose up -d
```

#### 5. 验证安装

```bash
curl http://localhost:8080/health
```

预期响应：
```json
{"status":"healthy","timestamp":"2026-03-18T12:00:00Z"}
```

#### 6. 访问界面

- **主应用**: http://localhost:8080
- **管理后台**: http://localhost:8080/admin

## 系统架构

### 数据流图

```
图片库(library/)
    ↓
扫描服务 → 数据库(SQLite)
    ↓
缩略图服务 → 缩略图存储
    ↓
AI 标签服务 → 标签治理 → 数据库
    ↓
Flutter Web 前端 ← API 服务
```

### 核心数据表

| 表名 | 用途 |
|------|------|
| `images` | 图片元数据（路径、尺寸、格式、哈希） |
| `tags` | 标准标签（用户确认的规范标签） |
| `tag_aliases` | 标签别名映射 |
| `tag_observations` | AI 原始标签观测记录 |
| `image_tags` | 图片与标签的关联 |
| `collections` | 收藏夹定义 |
| `collection_images` | 收藏夹与图片的关联 |
| `duplicate_groups` | 重复图片组 |
| `async_jobs` | 异步任务队列 |

## 功能使用指南

### 图片扫描和导入

#### 方式一：自动监控

系统启动后会自动监控配置的 `library/` 目录。将图片放入该目录，系统会自动扫描并导入。

#### 方式二：手动触发扫描

通过管理后台或 API 手动触发扫描：

**管理后台**:
1. 访问 http://localhost:8080/admin
2. 点击"立即扫描"按钮

**API 调用**:
```bash
curl -X POST http://localhost:8080/admin/api/actions/scan
```

#### 支持的图片格式

- JPG/JPEG
- PNG
- WebP
- GIF

#### 导入流程

1. **文件扫描**: 递归扫描指定目录
2. **元数据提取**: 提取尺寸、格式、EXIF 信息
3. **哈希计算**: 计算 SHA256 文件哈希和感知哈希
4. **缩略图生成**: 生成多种尺寸的缩略图
5. **数据库存储**: 保存图片元数据

### 标签管理

#### 标签体系

系统采用三层标签架构：

1. **标准标签（Tags）**: 用户确认的规范标签，用于最终检索
2. **别名（Aliases）**: 标准标签的变体表达（如"初音ミク"→"初音未来"）
3. **AI 观测（Observations）**: AI 生成的原始标签，待审核状态

#### 标签分类

系统支持以下标签分类：

- **人物关系**: 角色、CP、互动关系
- **服装外观**: 服饰、发型、配饰
- **姿态动作**: 姿势、表情、动作
- **场景环境**: 室内、室外、季节、时间
- **氛围情绪**: 氛围、情感、风格
- **美术风格**: 画风、技法、色彩
- **道具元素**: 武器、道具、特效
- **内容分级**: 年龄分级、内容类型

#### 添加标签

**方式一：通过图片详情页**
1. 打开图片详情
2. 点击"添加标签"
3. 输入标签名称或选择现有标签
4. 确认添加

**方式二：标签管理页面**
1. 访问标签管理界面
2. 点击"新建标签"
3. 填写标签名称和分类
4. 保存

#### 编辑标签

1. 进入标签管理页面
2. 找到需要编辑的标签
3. 点击编辑按钮
4. 修改名称、分类或别名
5. 保存更改

#### 删除标签

⚠️ **注意**: 删除标签会同时删除所有关联关系，请谨慎操作。

#### 标签别名管理

1. 选择标准标签
2. 点击"管理别名"
3. 添加别名（如为"Reimu Hakurei"添加别名"博丽灵梦"）
4. 系统会自动将别名归并到标准标签

### AI标签生成

#### 触发 AI 标签

**单张图片**:
1. 打开图片详情页
2. 点击"AI 分析"按钮
3. 等待处理完成

**批量处理**:
1. 在画廊界面选择多张图片
2. 点击"批量 AI 标签"
3. 确认后系统会将任务加入队列

#### 查看 AI 标签状态

图片详情页显示 AI 处理状态：
- **待处理**: 已加入任务队列
- **处理中**: AI 正在分析图片
- **完成**: 已生成标签建议
- **失败**: 处理出错，可重试

#### 标签审核流程

1. **查看建议**: AI 生成后会显示建议标签列表
2. **确认标签**: 点击"确认"将 AI 标签转为标准标签
3. **修改标签**: 编辑标签名称后再确认
4. **拒绝标签**: 点击"忽略"删除该建议
5. **批量确认**: 可一次性确认多个标签

#### AI 标签治理

系统会自动处理：
- **精确匹配**: AI 标签与现有标准标签完全一致时自动关联
- **别名匹配**: AI 标签匹配某个标准标签的别名时，自动映射到标准标签
- **新建标签**: 无匹配时创建待审核的新标签

#### 限流机制

为避免 API 调用超限，系统实现了 Token Bucket 限流：
- 默认每分钟最多 60 次请求
- 超限请求会自动排队等待
- 可在配置中调整 `requests_per_minute`

### 重复图片检测

#### 检测方式

系统支持两种重复检测：

1. **完全重复（SHA256）**: 检测文件内容完全相同的图片
2. **相似图片（pHash）**: 检测视觉上相似的图片

#### 触发检测

**通过管理后台**:
1. 访问管理后台
2. 点击"检测重复图片"

**通过 API**:
```bash
curl -X POST http://localhost:8080/api/v1/duplicates/detect
```

#### 查看重复组

1. 访问重复图片管理界面
2. 系统会分组显示重复/相似图片
3. 每组显示相似度分数（0-100）

#### 处理重复图片

对于每个重复组，您可以选择：
- **保留**: 保留所有图片
- **删除**: 删除选中的重复图片
- **合并标签**: 将删除图片的标签合并到保留图片

#### 设置相似度阈值

在重复检测设置中调整：
- **高（90+）**: 仅检测非常相似的图片
- **中（70-90）**: 平衡检测（推荐）
- **低（50-70）**: 检测更多可能的相似图片

### 搜索功能

#### 文件名搜索

在搜索框输入文件名关键字：
- 支持模糊匹配
- 支持通配符 `*`
- 实时显示搜索结果

#### 标签搜索

1. 点击"筛选"打开标签筛选面板
2. 选择要搜索的标签（可多选）
3. 选择匹配模式：

4. 点击应用，显示筛选结果

#### 组合搜索

支持多种条件组合：
- 标签 + 文件名
- 多个标签组合
- 标签 + 收藏夹

#### 搜索结果排序

- **相关性**: 匹配度高的优先
- **时间**: 按导入时间排序
- **名称**: 按文件名排序

#### 以图搜图

1. 点击搜索框旁的"相机"图标
2. 选择或拖拽图片上传
3. 系统会显示视觉上相似的图片

## 配置指南

### 配置文件位置

`deploy/config/config.yaml`

### 配置项说明

#### 服务器配置

```yaml
server:
  host: "0.0.0.0"      # 监听地址，0.0.0.0 表示所有接口
  port: 8080           # 服务端口
  env: "production"    # 环境：development 或 production
```

#### 数据库配置

```yaml
database:
  type: "sqlite"                    # 数据库类型
  path: "/data/acgwarehouse.db"     # 数据库文件路径
  connection_string: ""             # PostgreSQL 连接字符串（预留）
```

#### 存储配置

```yaml
storage:
  scan_roots:
    - "/library"       # 图片库扫描路径（容器内路径）
```

#### AI 配置

```yaml
ai:
  provider: "qwen"                  # AI 提供商：qwen / doubao / openai
  api_key: "YOUR_API_KEY"           # API 密钥
  model: "qwen-vl-max"              # 模型名称
  requests_per_minute: 60           # 每分钟最大请求数
```

#### COS 配置（可选）

```yaml
cos:
  bucket_url: "https://your-bucket.cos.ap-guangzhou.myqcloud.com"
  secret_id: "YOUR_SECRET_ID"
  secret_key: "YOUR_SECRET_KEY"
```

### 环境变量覆盖

可以通过环境变量覆盖配置：

| 变量名 | 说明 |
|--------|------|
| `SERVER_HOST` | 服务器监听地址 |
| `SERVER_PORT` | 服务器端口 |
| `DATABASE_TYPE` | 数据库类型 |
| `DATABASE_PATH` | 数据库路径 |
| `AI_API_KEY` | AI 服务 API 密钥 |
| `TZ` | 时区 |

在 `docker-compose.yml` 中设置：

```yaml
environment:
  - AI_API_KEY=your-api-key-here
  - TZ=Asia/Shanghai
```

## 常见问题解答

### Q: 系统支持多少张图片？

A: 系统基于 SQLite，经测试可支持 10,000+ 张图片的流畅浏览。对于更大的库，建议关注性能报告中的优化建议。

### Q: AI 标签需要额外付费吗？

A: 是的，AI 标签功能需要调用千问或豆包的 API，会产生相应费用。请参考各服务商的定价。

### Q: 如何更换 AI 提供商？

A: 修改 `config.yaml` 中的 `ai.provider` 和 `ai.api_key`，然后重启容器：

```bash
docker compose restart
```

### Q: 图片存储在哪里？

A: 图片存储在宿主机 `library/` 目录，通过 Docker volume 挂载到容器。数据库在 `data/` 目录。

### Q: 如何备份数据？

A: 备份两个目录即可：

```bash
# 停止服务
docker compose stop

# 备份
cp -r data/ data-backup-$(date +%Y%m%d)/
cp -r library/ library-backup-$(date +%Y%m%d)/

# 重启服务
docker compose start
```

### Q: 支持哪些图片格式？

A: 支持 JPG、PNG、WebP、GIF 格式。

### Q: 如何更新到新版？

A: 
```bash
# 备份数据后
docker compose down
git pull origin master
docker compose up -d --build
```

### Q: 缩略图存储在哪里？

A: 缩略图默认存储在数据库中。如需使用腾讯云 COS，请配置 COS 选项。

### Q: 任务队列卡住了怎么办？

A: 
1. 访问管理后台查看任务状态
2. 检查 AI API 密钥是否有效
3. 尝试暂停再恢复队列
4. 重启容器：`docker compose restart`

## 故障排除

### 容器无法启动

**现象**: `docker compose up` 后容器立即退出

**排查步骤**:
1. 检查日志：`docker compose logs app`
2. 确认端口 8080 未被占用
3. 检查 `data/` 和 `library/` 目录权限
4. 验证配置文件格式：`cat deploy/config/config.yaml | head -20`

### 图片无法显示

**现象**: 画廊中图片显示为空白

**排查步骤**:
1. 检查图片是否在 `library/` 目录中
2. 确认文件权限正确
3. 检查缩略图是否生成
4. 查看浏览器开发者工具的网络请求

### AI 标签一直失败

**现象**: AI 标签任务状态显示"失败"

**排查步骤**:
1. 检查 API 密钥是否有效
2. 查看管理后台的错误日志
3. 检查网络连接
4. 确认未达到 API 调用限额
5. 尝试降低 `requests_per_minute` 配置

### 数据库损坏

**现象**: 查询报错或数据异常

**修复步骤**:
1. 停止容器：`docker compose stop`
2. 备份当前数据库：`cp data/acgwarehouse.db data/acgwarehouse.db.bak`
3. 检查数据库完整性：
   ```bash
   docker run --rm -v $(pwd)/data:/data nouchka/sqlite3 \
     sqlite3 /data/acgwarehouse.db "PRAGMA integrity_check;"
   ```
4. 如损坏严重，从备份恢复或重建数据库

### 性能问题

**现象**: 大图库浏览卡顿

**优化建议**:
1. 检查容器资源使用：`docker stats`
2. 增加 Docker 容器内存限制
3. 减少每页显示数量
4. 考虑按标签筛选减少数据量

### 重置到干净状态

⚠️ **警告**: 这会删除所有数据！

```bash
docker compose down
rm -rf data/*
rm -rf library/*
docker compose up -d
```

3. 查看性能报告了解系统限制
4. 参考部署文档的故障排除章节
