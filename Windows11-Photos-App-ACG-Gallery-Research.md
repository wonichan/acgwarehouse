## 第一部分：Windows 11 照片程序 UI 风格与布局

### 1. 视觉风格与设计语言

#### 1.1 材质系统（Material）
Windows 11 照片程序采用 **Fluent 2 设计系统**，主要使用以下材质：

| 材质类型     | 特性                             | 应用场景               |
| ------------ | -------------------------------- | ---------------------- |
| **Mica**     | 不透明材质，微妙地融合桌面背景色 | 主窗口背景（默认）     |
| **Mica Alt** | Mica 的强化变体，对比度更高      | 需要更强视觉深度的场景 |
| **Acrylic**  | 半透明磨砂玻璃效果               | 弹出菜单、浮层面板     |
| **Smoke**    | 半透明黑色遮罩                   | 模态对话框、遮罩层     |

**关键洞察**：Mica 材质性能优于 Acrylic，因为它只采样一次壁纸颜色，而非持续模糊处理。

#### 1.2 圆角设计
- 窗口圆角：8px 标准半径
- 按钮/卡片圆角：4-8px
- 图片缩略图：保持原始比例，容器圆角

#### 1.3 色彩方案
- **主题感知**：自动跟随系统明/暗模式
- **独立主题**：可单独设置应用主题（不依赖系统）
- **强调色**：采用 Windows 11 系统强调色

#### 1.4 字体系统
- **主字体**：Segoe UI Variable（可变字重）
- **字号层级**：
  - 标题（Title）：28px / 字重 600
  - 副标题（Subtitle）：20px / 字重 600
  - 正文（Body）：14px / 字重 400
  - 说明文字（Caption）：12px / 字重 400

#### 2.2 左侧导航栏（NavigationView）

**导航模式**：左侧展开式（Left Expanded）
- **宽度**：240px（展开状态）/ 48px（收起状态）
- **边距**：24px 内容边距

**导航层级**：
```

**模式一：瀑布流模式（River Mode）**
- 图片按原始宽高比显示
- 接近真实照片墙效果
- 适合浏览艺术作品、截图

**模式二：方块模式（Square Mode）**
- 统一方形缩略图
- 布局更整洁规范
- 适合快速浏览大量图片

**筛选功能**：
- 全部 / 仅照片 / 仅视频
- 按日期范围筛选

#### 2.4 查看器界面

**独立窗口设计**：
- 双击图片在新窗口打开
- 不阻塞主图库窗口操作
- 支持同时打开多个查看器

**查看器布局**：
```

**底部胶片条（Filmstrip）**：
- 显示当前文件夹内所有图片缩略图
- 可快速切换图片

### 4. 优缺点分析

#### 4.1 优点

1. **现代化界面**
   - Fluent Design 与 Windows 11 系统风格统一
   - Mica 材质营造层次感
   - 圆角设计柔和现代

2. **多视图模式**
   - River/Square 模式满足不同浏览偏好
   - 瀑布流模式适合艺术作品浏览

3. **独立查看窗口**
   - 非模态设计，支持多任务
   - 可同时打开多个图片对比

4. **胶片条设计**
   - 查看时可快速切换到同文件夹其他图片
   - 支持多图对比模式

5. **云原生设计**
   - OneDrive/iCloud 深度集成
   - 自动同步云端照片

#### 4.2 局限性

1. **功能缺失**
   - 已移除视频编辑器（需使用 Clipchamp）
   - 已移除人脸识别和人物分组
   - 不支持标签元数据搜索

2. **交互问题**
   - 单击选择 vs 双击打开 容易混淆
   - 查看器窗口大小不会记住用户设置

3. **版本分裂**
   - 部分功能需 Photos Legacy（旧版）
   - AI 功能需要特定硬件（Copilot+ PC）

## 第二部分：ACG 图库扩展功能方案

### 1. AI 自动标签生成

#### 1.1 功能目标
每张图片通过 AI 分析自动生成 5-8 个最相关的标签。

#### 1.2 技术实现参考

云端 API**
- 阿里千问系列、豆包AI
- 优点：识别准确率高
- 缺点：需联网、可能产生费用、隐私风险

#### 1.4 UX 设计

**自动生成流程**：
1. 图片导入时自动触发 AI 分析
2. 生成标签显示为"建议标签"（虚线边框）
3. 用户可一键接受全部或逐个确认
4. 用户可手动创建标签或编辑 AI 标签

**手动干预**：
- 右键图片 → "编辑标签"
- 输入时智能提示已有标签
- 支持批量应用标签

**基础筛选**：

### 4. 重复照片去重处理

#### 4.1 重复检测算法

**第一阶段：精确重复（File Hash）**
- 计算文件 MD5/SHA1 哈希
- 100% 相同文件快速识别
- 时间复杂度 O(n)

**第二阶段：视觉相似（Perceptual Hash）**
- 使用 pHash（感知哈希）算法
- 检测压缩、缩放后的相似图片
- 汉明距离 < 5 认为是相似图片

**第三阶段：AI 相似度（可选）**
- 使用 CLIP 嵌入向量比对
- 检测内容相似但视觉差异大的图片
- 计算余弦相似度

#### 4.2 重复处理界面

```
┌─────────────────────────────────────────────────────┐
│  重复图片检测                          [重新扫描]    │
├─────────────────────────────────────────────────────┤
│                                                      │
│  发现 12 组重复图片（共 34 张）                       │
│                                                      │
│  ┌─────────────────────────────────────────────┐   │
│  │ 第 1 组（3 张）    [保留最大] [删除其他]       │   │
│  │                                              │   │
│  │  [图片A 2.3MB]  [图片B 1.8MB]  [图片C 2.3MB] │   │
│  │  1920×1080      1280×720       1920×1080     │   │
│  │  ✓ 保留         ✗ 删除         ✗ 删除        │   │
│  └─────────────────────────────────────────────┘   │
│                                                      │
│  ┌─────────────────────────────────────────────┐   │
│  │ 第 2 组（2 张）    [保留最大] [删除其他]       │   │
│  │  ...                                         │   │
│  └─────────────────────────────────────────────┘   │
│                                                      │
│  [应用选中的操作]                                    │
└─────────────────────────────────────────────────────┘
```

#### 4.3 智能选择策略

| 策略         | 说明                         |
| ------------ | ---------------------------- |
| 保留最大     | 保留分辨率最高的版本         |
| 保留最新     | 保留修改日期最新的版本       |
| 保留最完整   | 保留 EXIF 信息最完整的版本   |
| 保留最佳质量 | 通过模糊检测保留最清晰的版本 |

#### 4.4 处理模式

- **查看模式**：手动选择保留哪张
- **自动模式**：按策略自动处理（推荐先查看再应用）
- **堆叠模式**：将重复图片堆叠显示，主图显示在最上层

## 总结

本方案基于 Windows 11 照片程序的 UI 风格和交互模式，针对 ACG 图库的特殊需求进行了以下核心扩展：

1. **AI 自动标签**：针对 ACG 内容优化，支持角色、作品、场景等多维度标签
2. **标签筛选系统**：从简单的标签点击扩展到布尔逻辑和保存筛选预设
3. **标签管理**：引入层级结构、别名、颜色标识等组织功能
4. **重复检测**：从简单的文件哈希扩展到视觉相似度检测
